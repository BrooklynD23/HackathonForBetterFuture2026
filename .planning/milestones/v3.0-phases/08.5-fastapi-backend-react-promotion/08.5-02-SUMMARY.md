# Phase 8.5 Plan 02: React Promotion + API Client Summary

V1.1 Figma-exported React mockup promoted to a standalone production-ready Vite app at `frontend/`, with all 7 pages rendering and a typed API client wired to the FastAPI backend on port 8000.

## What Was Built

### Task 1: Promote V1.1 Mockup to frontend/

The V1.1 mockup from `docs/mockup/V1.1/IA-West_UI/` was promoted to `frontend/` as a fully standalone React application:

- **package.json** rewritten from scratch — changed name from `@figma/my-make-file` to `ia-west-smartmatch-frontend`, moved react/react-dom from peerDependencies to dependencies, added dev/build/preview scripts, added TypeScript devDependencies, removed pnpm overrides and peerDependenciesMeta sections
- **vite.config.ts** updated with API proxy (`/api` → `http://localhost:8000`), port 5173, path alias `@` → `./src`, and asset includes for SVG/CSV
- **tsconfig.json** created with react-jsx, bundler module resolution, strict mode, and path alias
- **index.html** created with root div entry point and module script
- **src/main.tsx** created with ReactDOM.createRoot bootstrap
- All 7 page components promoted from mockup: Dashboard, AIMatching, Calendar, Opportunities, Outreach, Pipeline, Volunteers
- 50+ shadcn/ui component files (accordion, button, dialog, form, select, etc.) promoted intact

Build result: `npx vite build` exits 0 with a 719 kB bundle (non-fatal chunk size warning only).

### Task 2: Typed API Client Module

`frontend/src/lib/api.ts` provides typed fetch wrappers for all 9 FastAPI endpoints:

**Data endpoints:**
- `fetchSpecialists()` → `Specialist[]`
- `fetchEvents()` → `CppEvent[]`
- `fetchPipeline()` → `PipelineRecord[]`
- `fetchCalendar()` → `CalendarRecord[]`
- `fetchContacts()` → `PocContact[]`

**Matching endpoints:**
- `rankSpeakers(eventName, limit?)` → `RankedMatch[]` (with field normalization)
- `scoreSpeaker(speakerName, eventName)` → `MatchScore`

**Outreach endpoints:**
- `generateEmail(speakerName, eventName)` → `OutreachEmailResponse`
- `generateIcs(eventName, eventDate?, location?, description?)` → `{ ics_content: string }`

The `requestJson<T>` helper extracts FastAPI `detail` error strings for user-friendly error messages. `normalizeRankedMatch` handles field name variance between ranking endpoint response shapes.

- `Dashboard.tsx` — hardcoded metric cards and activity list

These stubs do NOT prevent Plan 02's goal (a working dev server + proxy config + typed API client). They will be wired in Phase 9 plans.
