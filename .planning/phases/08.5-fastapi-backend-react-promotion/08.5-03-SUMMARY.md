# Plan 03 Summary: Wire React Pages to FastAPI

## What Changed

All 7 data-driven React pages were rewired from hardcoded mock data to live API calls via the `@/lib/api` module created in Plan 02.

### Pages Wired

| Page | API Endpoints Used | Key Features |
|------|-------------------|--------------|
| Volunteers | `fetchSpecialists` | Grid with search/filter, detail modal, expertise tags |
| Pipeline | `fetchPipeline`, `fetchEvents` | Funnel chart (Recharts), university breakdown, stage counts |
| Dashboard | `fetchSpecialists`, `fetchEvents`, `fetchPipeline`, `fetchCalendar`, `rankSpeakers` | 4 metric cards, funnel, bar chart, line chart, top matches |
| AI Matching | `fetchEvents`, `rankSpeakers`, `generateEmail` | Event selector, ranked results with factor scores, email gen |
| Calendar | `fetchCalendar` | Month/list views, IA event filtering, nearby universities |
| Outreach | `fetchEvents`, `rankSpeakers`, `generateEmail` | Template editor, email generation, send simulation |
| Opportunities | `fetchEvents` | Event grid with search, category/audience filters |

### Shared Pattern

Every page follows the same async data pattern:
```typescript
const [data, setData] = useState<T[]>([]);
const [loading, setLoading] = useState(true);
const [error, setError] = useState<string | null>(null);

useEffect(() => {
  let active = true;
  fetchData()
    .then(d => active && setData(d))
    .catch(e => active && setError(e.message))
    .finally(() => active && setLoading(false));
  return () => { active = false; };
}, []);
```
