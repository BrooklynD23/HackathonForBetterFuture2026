## Component Inventory

New components added in Phase 7:

### 1. Swimlane Dashboard (`src/ui/swimlane_dashboard.py`)

**Layout:** `st.columns(N)` horizontal row — one column per dispatched agent (max 8).

**Card anatomy (per swimlane card):**
```
┌─────────────────────────────┐  border: 1px solid {status-color}
│ Agent Name (Inter Tight 600)│  border-radius: 8px
│ STATUS BADGE  (13px, upper) │  padding: 8px (md inner)
│ 12s elapsed   (13px, 400)   │  background: var(--surface-container-lowest)
│ Result summary (13px, 400)  │
│ [Retry]  (only on failed)   │
└─────────────────────────────┘
```

**Retry button:** `st.button("Retry", key=f"retry_{proposal_id}")` — secondary type. Appears only when status is "failed".

**Collapse behavior:** Cards with status "completed" older than 30 seconds collapse to single-line text showing agent name + "Done" in green. Full card shown for all other statuses at all times.

**Visibility rule:** Section heading "### Agent Status" and the columns grid render only when `agent_swimlanes` session state dict is non-empty. No placeholder or skeleton shown when empty.

**Section position:** Inserted in `render_command_center_tab()` between the voice panel div and the `st.divider()` before conversation history. Called inside `_poll_result_bus()` fragment to auto-refresh every 2 seconds.

Source: CONTEXT.md swimlane design decision + RESEARCH.md Pattern 2.

### 2. Demo Hint Chips (inline in `src/ui/command_center.py`)

**Trigger:** Rendered only when `conversation_history` session state list is empty.

**Layout:** Three `st.button()` calls in a single `st.columns(3)` row.

**Labels (exact, do not change):**
1. "Find new events"
2. "Rank speakers for CPP Career Fair"
3. "Prepare full outreach campaign"

**Interaction:** Clicking a chip calls `_handle_text_command(label)` exactly as if the user typed it. No special styling beyond Streamlit default secondary button style.

**Position:** Rendered after voice panel, before swimlane dashboard section.

Source: CONTEXT.md TTS/Demo Polish decision.

### 3. TTS Completion Playback (extension to `_poll_result_bus()`)

**Trigger:** When `poll_results()` returns a payload with `status: "completed"`, call `_speak_text()` with the formatted result string.

**Multi-agent summary format:** One sentence per completed agent in the same poll cycle, concatenated with a space separator. Example: "Discovery found 12 events. Speaker ranking complete for 8 speakers."

No new UI component — extension of existing `_speak_text()` and `_poll_result_bus()`.

Source: CONTEXT.md TTS Result Summaries decision.

### 4. POC Overdue Suggestion Card (extension to `_inject_proactive_suggestions()`)

**Visual:** Identical to existing proactive suggestion action cards (uses `_render_action_card()` unchanged).

**Suggestion message format:** "{N} contact(s) overdue for follow-up: {name1}, {name2}, {name3} — review now?"

**Priority rule:** Rendered only when no staleness suggestion was already injected in the same call. One proactive suggestion maximum at a time (existing guard).

No new UI component — reuses `_render_action_card()` and existing HITL approve/reject flow.

Source: CONTEXT.md POC Proactive Suggestions decision.

## Interaction States

### Swimlane Card State Machine

```
idle → running → completed
                → failed → [Retry button shown] → running (on retry click)
idle → running → awaiting → running → completed
                                    → failed
```

- State transitions driven by `agent_swimlanes` session state dict, updated by `_poll_result_bus()` fragment every 2 seconds
- Cards do NOT animate between states — border color change is the only state indicator
- "Collapse to compact" (completed + 30s elapsed) is a visual simplification only — card remains in session state

### Demo Hint Chips Visibility

- Visible: `len(st.session_state.get("conversation_history", [])) == 0`
- Hidden: any conversation history entry exists
- Hidden state is permanent for the session (once hidden, never re-shown)

### Proactive Suggestion Injection (POC-03)

- Runs once per `render_command_center_tab()` call
- Guard: if any proactive suggestion is already pending (source == "proactive" in action_proposals), skip injection
- Staleness check runs first; if no staleness suggestion injected AND overdue contacts exist, inject POC suggestion
- Max 1 new injection per render call
