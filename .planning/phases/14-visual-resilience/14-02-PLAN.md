**3. Update `frontend/src/lib/api.ts` fetch functions:**

Add a generic `WithSource<T>` wrapper type:

```typescript
export interface WithSource<T> {
  data: T;
  source: "live" | "demo";
}
```

Update these fetch functions to detect `"source": "demo"` from the backend response:

For **list-returning endpoints** (`fetchPipeline`, `fetchSpecialists`, `fetchEvents`, `fetchCalendar`, `fetchCalendarEvents`, `fetchCalendarAssignments`):

The backend injects `"source": "demo"` per-row. Read it from `payload[0]?.source`. Return `WithSource<T[]>`:

```typescript
export async function fetchPipeline(): Promise<WithSource<PipelineRecord[]>> {
  const raw = await requestJson<unknown>("/api/data/pipeline");
  const payload = toRecordArray(raw);
  const source: "live" | "demo" = (payload[0] as Record<string, unknown>)?.source === "demo" ? "demo" : "live";
  return {
    data: payload.map((item, i) => normalizePipelineRecord(item, i)),
    source,
  };
}
```

Apply same pattern to `fetchSpecialists`, `fetchEvents`, `fetchCalendar`, `fetchCalendarEvents`, `fetchCalendarAssignments`. Each must read `source` from `payload[0]?.source` before normalizing.

For **dict-returning endpoints** (`fetchQrStats`, `fetchFeedbackStats`):

The backend injects `"source": "demo"` as top-level field. Read it before normalization:

```typescript
export async function fetchFeedbackStats(): Promise<WithSource<FeedbackStatsSummary>> {
  const raw = await requestJson<Record<string, unknown>>("/api/feedback/stats");
  const source: "live" | "demo" = raw?.source === "demo" ? "demo" : "live";
  return {
    data: normalizeFeedbackStats(raw),
    source,
  };
}
```

Apply same pattern to `fetchQrStats`.

**Important:** The return type changes from `Promise<T>` to `Promise<WithSource<T>>`. Every call site must destructure `{ data, source }`. This is intentional — the call sites are updated in Task 2.
