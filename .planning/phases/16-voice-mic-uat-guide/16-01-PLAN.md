**Title:** `# Voice/Mic UAT Guide -- IA West SmartMatch CRM`

**Section 1: `## Overview`**
- One paragraph: This guide enables a reviewer to independently walk through the live voice/mic coordinator path, verifying intent parsing, human approval gating, and agent action execution. No prior implementation knowledge required.

**Section 2: `## Prerequisites`**
- Numbered checklist of requirements before starting:
  1. Python 3.10+ installed with project dependencies (`pip install -r requirements.txt`)
  2. `streamlit-webrtc` installed (verify: `python -c "import streamlit_webrtc; print('OK')"`) -- if missing, the UI shows: `"streamlit-webrtc not installed. Use text commands."`
  3. Browser: Chrome or Chromium recommended (Firefox may block WebRTC audio)
  4. Microphone: Built-in or external mic connected (only needed for Path A: Voice Input)
  5. Gemini API key (optional): Set `GEMINI_API_KEY` env var for LLM intent parsing. Without it, keyword fallback is used -- both produce identical approval cards. The reasoning field will show `"Matched keyword in: '...'"` instead of an LLM explanation.

**Section 3: `## Starting the Application`**
- Step-by-step startup:
  1. `cd "Category 3 - IA West Smart Match CRM"`
  2. `streamlit run src/app.py`
  3. Open browser to `http://localhost:8501`
  4. Note: FastAPI backend (`:8000`) is NOT required for the voice/mic path
  5. Expected on first load: spinners reading `"Loading voice model (first use)..."` and `"Loading speech recognition..."`. Wait for both to complete.
  6. Navigate to the Command Center tab (2nd tab in the Coordinator Dashboard section)
  7. Expected initial state: heading reads `"Jarvis -- Voice Command Center"`, empty conversation area shows `"No conversation yet"` with three demo hint chips: `"Find new events"`, `"Rank speakers for CPP Career Fair"`, `"Prepare full outreach campaign"`

**Section 4: `## Path A: Voice Input`**
- Numbered steps, each with **Action:** and **Expected:** lines:
  1. Action: Click the microphone button (right side of voice panel, narrow column next to the text input).
     Expected: Browser shows a microphone permission prompt (Chrome: small popup near address bar saying "localhost:8501 wants to use your microphone").
  2. Action: Click "Allow" in the browser permission prompt.
     Expected: The WebRTC widget shows an active state (START/STOP button becomes active). The mic is now recording.
  3. Action: Speak clearly into the microphone: "Find new events". Then click the STOP button on the WebRTC widget to end recording.
     Expected: A spinner appears reading `"Transcribing..."`. After 2-5 seconds, the transcript appears in the conversation history as a user message, and an approval card appears below it.
  4. Action: Examine the approval card.
     Expected: The card shows:
     - Agent name: **Discovery Agent**
     - Status: `proposed`
     - Description: "Scrape universities for new events"
     - Reasoning line (either LLM explanation or `"Matched keyword in: 'find new events'"`)
     - Two buttons: **Approve** (blue/primary) and **Reject**
  5. Action: Click the **Approve** button on the Discovery Agent card.
     Expected: Status changes from `proposed` to `executing` (briefly shows `"Status: executing..."`), then transitions to `completed` with a green success box showing a result like `"Found N event(s) (source: ...)"`.
  6. Action: Observe the conversation history scroll area.
     Expected: The completed action card remains in the conversation history with status `completed` and the result text visible.

**Section 5: `## Path B: Text Input (Fallback)`**
- Numbered steps, each with **Action:** and **Expected:** lines:
  1. Action: In the text input field (placeholder reads `"Type a command or use the mic..."`), type: `Rank speakers for CPP Career Fair`
  2. Action: Click the **Send Command** button (blue/primary button below the text input).
     Expected: A new user message appears in conversation history, followed by an approval card.
  3. Action: Examine the approval card.
     Expected: The card shows:
     - Agent name: **Matching Agent**
     - Status: `proposed`
     - Description: "Rank speakers for a target event"
     - Reasoning line with LLM or keyword explanation
     - **Approve** and **Reject** buttons
  4. Action: Click the **Reject** button.
     Expected: The card updates to show an orange/yellow warning: `"Action rejected by coordinator."` The status changes to `rejected`. No agent execution occurs.

**Section 6: `## Multi-Step Workflow: Prepare Campaign`**
- Numbered steps:
  1. Action: Type `Prepare full outreach campaign` in the text input and click **Send Command**. (Alternatively, click the `"Prepare full outreach campaign"` demo hint chip if conversation is empty.)
     Expected: THREE approval cards appear in succession (not one), because `prepare_campaign` is a multi-step intent that spawns 3 sub-proposals.
  2. Action: Examine the three approval cards.
     Expected: The cards are:
     - **Discovery Agent** -- "Scrape universities for new events" -- Status: `proposed`
     - **Matching Agent** -- "Rank speakers for a target event" -- Status: `proposed`
     - **Outreach Agent** -- "Draft outreach emails for a match" -- Status: `proposed`
     Each has independent Approve/Reject buttons. The reasoning for all three reads: `"Part of 'Prepare full outreach campaign' campaign orchestration."`
  3. Action: Approve one card (e.g., Discovery Agent) and reject another (e.g., Outreach Agent). Leave the third as `proposed`.
     Expected: The approved card transitions through `executing` to `completed`. The rejected card shows `"Action rejected by coordinator."` The third card remains in `proposed` state with Approve/Reject buttons still active. This demonstrates independent approval gating per sub-action.

**Section 7: `## Edge Cases`**

Sub-section `### Microphone Permission Blocked`:
- Symptom: No mic permission prompt appears, or mic button does not activate.
- Fix: In Chrome, click the lock/info icon in the address bar > Site settings > Microphone > set to "Allow". Or navigate to `chrome://settings/content/microphone` and remove `localhost:8501` from the Blocked list. Reload the page.

Sub-section `### STT Model Load Failure`:
- Symptom: Yellow warning appears: `"Speech recognition unavailable. Please use text commands."`
- Cause: faster-whisper model download failed or dependency missing.
- Handling: Use Path B (text input) instead. All approval card behavior is identical regardless of input method.

Sub-section `### No Speech Detected`:
- Symptom: After stopping mic recording, warning appears: `"No speech detected. Please try again."`
- Fix: Re-press the mic button, speak closer to the microphone, and check OS microphone input level settings. Ensure Chrome has the correct microphone device selected if multiple are available.

Sub-section `### streamlit-webrtc Not Installed`:
- Symptom: Warning appears in mic column: `"streamlit-webrtc not installed. Use text commands."`
- Fix: Install with `pip install streamlit-webrtc` and restart the Streamlit app. Until installed, use Path B (text input).

Sub-section `### Gemini API Key Not Set (Keyword Fallback)`:
- Symptom: Approval cards still appear, but reasoning reads `"Matched keyword in: '...'"` instead of a natural-language LLM explanation.
- Impact: None on functionality. Intent parsing, approval cards, and agent execution work identically. The only difference is the reasoning text.

Sub-section `### Unknown Command`:
- Symptom: After sending a command, instead of an approval card, Jarvis replies: `"I couldn't understand that command. You said: '{text}'. Try rephrasing -- for example, 'find new events' or 'rank speakers for event X'."`
- Handling: Rephrase using one of the supported command patterns (see Supported Commands Reference below).

**Section 8: `## Supported Commands Reference`**
- Table with columns: Intent | Agent | Sample Commands | Description
- Rows:
  - `discover_events` | Discovery Agent | "Find new events", "Discover events", "Scrape for events" | Scrape universities for new events
  - `rank_speakers` | Matching Agent | "Rank speakers for CPP Career Fair", "Who should speak at X?" | Rank speakers for a target event
  - `generate_outreach` | Outreach Agent | "Draft outreach email", "Generate email" | Draft outreach emails for a match
  - `check_contacts` | Contacts Agent | "Check contacts", "Follow up on POCs" | Review POC contact status
  - `prepare_campaign` | Campaign Orchestrator | "Prepare full outreach campaign", "Launch campaign" | Multi-step: discover + rank + outreach (3 sub-proposals)
