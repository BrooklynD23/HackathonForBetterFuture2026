# Demo Script: IA West SmartMatch CRM

## 1. Demo Philosophy

Three principles govern this demonstration. First, **show, don't tell** -- every claim in the Growth Strategy, Measurement Plan, and Responsible AI Note is visible on screen. Second, **name real people and real events** -- "Travis Miller matched to UCLA's hackathon" is verifiable; "Speaker A matched to Event B" is not. Third, **end on the funnel** -- the last thing judges see is the pipeline visualization, because the pipeline is what IA West has never had.

---

## 3. Tab Tour Guide

After the 60-second moment, walk judges through each tab at the pace below. Total tour time: approximately 5 minutes.

| Tab | Duration | Must Show | Connects To |

## 4. Narrative Bridges

Use these scripted transitions to connect the live demo back to the written deliverables. Deliver each at the natural pause point between tabs.

**Demo to Growth Strategy:**
"What you just saw with UCLA is Phase 2 of our rollout. Phase 1 is already loaded -- 15 CPP events, 35 courses, all matched. Phase 3 extends this to all 5 pre-configured West Coast universities. Adding UCLA took 3 steps: paste URL, AI extracts events, confirm. Any volunteer metro director can do this for their local universities without technical training."

**Demo to Measurement Plan:**
"That accept/decline button Travis would click feeds directly into our feedback loop. Each decline captures a structured reason -- 'topic mismatch,' 'too far,' 'schedule conflict' -- mapped to the exact scoring factor it should adjust. After 30 decisions, the system suggests weight adjustments. It's A/B testing your own matching algorithm, with the same statistical rigor you'd apply to a market-research field study."

**Demo to Responsible AI:**
"Notice every match shows 8 named factors, not a black-box score. Travis can see exactly why he was recommended, and chapter leadership can override any weight they disagree with. We scrape only public event pages -- no login-gated content, no student PII, rate-limited at 1 request per 5 seconds. The system contains zero student records."

**Demo to Scalability:**
"Adding UCLA took 3 steps -- paste URL, AI extracts events, confirm. The same workflow scales to any public university events page. The matching algorithm runs across all 18 board members in under 60 seconds. CSV-based storage means no cloud database dependencies. The entire system runs locally on chapter infrastructure."

## 5. Failure Contingency

Prepare for each scenario before presentation day. Test all fallbacks end-to-end during rehearsal.

| Failure | Fallback | Preparation |
|---------|----------|-------------|
| **Scraping fails during demo** | Switch to Demo Mode -- pre-cached results for all 5 universities (UCLA, SDSU, UC Davis, USC, Portland State) load instantly with identical UI | Pre-warm cache for all 5 universities the morning of presentation; verify cached event cards render correctly |
| **Gemini API down** | Demo Mode serves cached email text and cached explanation cards; weight sliders and radar charts still function (local computation) | Verify all demo fixtures are populated; test with network adapter disabled |
| **Streamlit crashes** | Play recorded backup video of the full 60-second moment; narrate live over the recording | Record video in advance at 1080p; keep video file queued on a second device |
| **Network down entirely** | Local-only mode with all cached data -- no live scraping, but matching, pipeline, and dashboard tabs fully functional from CSV data | Test offline mode end-to-end; confirm all tabs render without network access |

**Pre-Presentation Checklist:**
- [ ] Cache pre-warmed for all 5 universities
- [ ] Demo Mode toggle verified
- [ ] Backup video recorded and tested on secondary device
- [ ] Offline mode tested with Wi-Fi disabled
- [ ] All 77 records loaded in CSV data files
- [ ] Gemini API key valid and rate limits confirmed
