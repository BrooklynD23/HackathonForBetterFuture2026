# Prototype: Simulated Market Research Pipeline

**Aytm x Neo Smart Living Joint Challenge — CPP AI Hackathon 2026**

This is a working prototype of the end-to-end simulated market research pipeline described in the challenge document. It demonstrates how LLMs can simulate qualitative interviews and quantitative survey responses for a real business case (Neo Smart Living's Tahoe Mini backyard prefab unit).

**You don't need a marketing background to use this code.** This README explains what each piece does and why.

---

## What Is This? (The 2-Minute Version)

Market research usually works like this:

1. **Qualitative phase** — Interview 20-30 real people to understand their needs, emotions, and language (costs $3K-$9K)
2. **Quantitative phase** — Survey 300-600 people with structured questions to measure preferences at scale (costs $6K-$24K for niche audiences)

This prototype replaces both phases with LLM-generated "synthetic respondents" — AI personas that role-play as target customers and answer interview/survey questions. Total cost: ~$2-$4 in API calls.

**The catch:** Synthetic data has known distortions (see the challenge document). Your job is to improve the pipeline — make it more reliable, more transparent, or more accessible.

## Quick Start (No API Key Needed)

The prototype includes pre-generated sample data so you can explore the dashboards immediately:

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Download NLTK data (needed for interview analysis)
python -c "import nltk; nltk.download('vader_lexicon'); nltk.download('punkt_tab'); nltk.download('stopwords')"

# 3. Explore the qualitative interview dashboard
streamlit run interview_dashboard.py

# 4. Explore the quantitative survey dashboard
streamlit run dashboard.py

# 5. Generate a static report (CSVs + charts)
python report.py
```

The sample data in `output/` was generated by the prototype — you can explore it without making any API calls.

## Why Two LLMs?

The prototype uses **GPT-4.1-mini** and **Gemini 2.5 Flash** to generate the same types of responses. This is a cross-model reliability check:

- If both models produce similar results on a variable, that finding is more trustworthy
- If models diverge significantly (Mann-Whitney U, p < 0.05), the finding should be flagged as uncertain
- The dashboards include a **Model Comparison** tab that highlights where models agree and disagree

This dual-LLM design is based on the STAMP methodology (Lin, under review) — see the challenge document for details.

## Marketing Research Concepts (For Non-Marketing Participants)

| Term | What It Means | Where You See It |
|---|---|---|
| **Qualitative research** | Open-ended interviews to understand *why* people think/feel/behave a certain way | Phase 1 (interview transcripts) |
| **Quantitative research** | Structured surveys with numeric scales to measure *how many* and *how much* | Phase 2 (survey responses) |
| **Likert scale (1-5)** | A rating scale from "Strongly Disagree" to "Strongly Agree" (or similar) | Most survey questions (Q1, Q2, Q5, Q7, Q9-Q17, Q19) |
| **Market segments** | Groups of customers with similar needs/behaviors | 5 segments in `segments.py` |
| **Concept test** | Showing different product positioning messages and measuring which resonates most | Q9-Q14 (5 positioning concepts) |
| **Sentiment analysis** | Classifying text as positive/negative/neutral | VADER scores in interview analysis |
| **LDA (topic modeling)** | Algorithm that discovers recurring themes across many text documents | Used on interview transcripts |
| **Mann-Whitney U test** | Statistical test comparing two groups (here: GPT vs. Gemini responses) | Model Comparison tabs |
| **Persona** | A fictional but realistic profile of a target customer | 30 in `interview_personas.py`, 5 segment archetypes in `segments.py` |
| **Digital twin** | An AI-generated stand-in for a real person (used in recent research) | What this entire prototype creates |

## Output Data Schema

### Interview Transcripts (`output/interview_transcripts.csv`)

30 rows. Key columns: `persona_id`, `model`, `age`, `income`, `work_arrangement`, `IQ1`-`IQ8` (interview responses), `additional_thoughts`

### Interview Analysis (`output/interview_analysis.csv`)

30 rows. Adds: `sentiment_IQ1`-`sentiment_IQ8`, `sentiment_overall`, `sentiment_label`, `primary_emotion`, `secondary_emotion`, `emotion_intensity`, `emotion_reasoning`

### Survey Responses (`output/synthetic_responses.csv`)

60 rows. Key columns: `respondent_id`, `model`, `segment_name`, `Q1`-`Q30` (survey answers). See `analytics.py` for column group constants (`LIKERT_KEYS`, `BARRIER_KEYS`, `CONCEPT_APPEAL`, `DEMOGRAPHIC_KEYS`).
